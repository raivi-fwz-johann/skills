# 设计阶段

[目标]
在编码前明确定义范围、约束与验证策略。

[准入条件]
- 读取 `TASK_STATE.md` (获取上下文)。

[退出条件]
- 明确意图与验收标准。
- 生成推荐图示清单。
- 更新 `TASK_STATE.md`。

[执行步骤]
1. 评估当前意图与现有架构的冲突情况。
2. 输出: 目标、影响范围、依赖关系、验收标准。
3. 输出: 推荐图示清单 (参见 `[图示推导规则]`)。
4. 等待人工确认 (设计阶段绝对不可自动跳过)。

[图示推导规则]
- C4 上下文/容器图: 出现新的宏观边界或外部服务。
- 类图: 核心领域模型、接口契约设计、多态机制。
- 活动图: 跨泳道业务流、并发/分叉 (Fork/Join) 处理。
- 序列图: 跨组件 API 调用、异步握手流。
- 状态图: 核心实体 (如指令状态、总线状态) 的生命周期。
- 流程图: 复杂的条件判断分支或算法逻辑。
- 产物依赖图: 包链接、CMake 编译依赖、环形依赖排查。

[禁止事项]
- 在本阶段过早进行深层级的细节编码。
- 悄无声息地擅自修改未受本次任务影响的关联模块。
