# 实现阶段

[目标]
完成一个独立的功能小步代码实现，为后续检视与验证做准备。

[准入条件]
- 设计与画图阶段已完成。
- 已加载 `ai-task/TASK_STATE.md` 中对应的目标任务区块。

[退出条件]
- 步长 N 编码完成。配套测试骨架已创建。
- 更新 `ai-task/TASK_STATE.md`。
- 准备好进入检视阶段。

[执行步骤]
1. 提出第 N 步的具体实现方案。(自动模式下由 AI 内部自检；手动模式下请求人工批准)。
2. 生成业务代码与测试骨架。
3. 针对 Lint 报错执行自动修复 (最大重试: 2 次)。
4. 自动模式 -> 放行至检视阶段。手动模式 -> 等待人工确认。

[质量底线]
- [必须] 严格遵循单一职责 (SRP)、KISS 及开闭原则。
- [必须] 命名需完全自我解释。注释只允许写“为什么(WHY)”，严禁描述“做什么(WHAT)”。
- [必须] 全要素防御性编程 (空指针判空、边界与异常处理)。
- [必须] 强制同步创建测试用例 (绝不允许“先堆代码后补测试”)。
- [禁止] 夹带与当前小步目标无关的任何重构。
- [禁止] 遗留仅用于临时跑通的过渡性废代码。

[升级人工条件]
- 发现原设计在物理实现层面不可落地。
- 必须进行跨越当前模块的大规模重构，超出受控范围。
- 自动修复报错连续失败 >= 2 次。
