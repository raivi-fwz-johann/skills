# 全局工作流状态与约束

[流转顺序]
标准通道: 设计 -> 画图 -> 实现 -> 检视 -> 验证
敏捷通道 (仅限微小修复): 实现 -> 自动检视 -> 验证

[执行循环: 实现/检视/验证]
- 粒度: 每次仅执行一个功能小步。
- 循环: 实现(步N) -> 检视(步N) -> 验证(步N) -> 确认通过 -> 推进(步N+1)。

[状态管理: ai-task/TASK_STATE.md]
- [必须] 进入任何阶段前必须先读取。
- [必须] 退出任何阶段前必须写入更新。
- 多任务处理: 仅更新当前活跃的区块（如 `# 目标 1`）。
- 归档: 将已完成的区块移至 `ai-task/TASK_ARCHIVE.md`。

[升级人工条件] (即使在自动模式下)
- 自动修复失败 >= 2 次。
- 存在多个等效路径 (AI 无法做出明确最优决策)。
- 发现设计与代码实现存在结构性冲突。
- 涉及高风险或不可逆操作。

[禁止事项]
- 跳过设计阶段直接编码（进入敏捷通道除外）。
- 在验证通过前，累积堆叠多个实现小步。
- 遗留仅用于临时跑通的过渡性兼容代码。
